# UI Design System Guidelines

This project uses a custom design system built with **Tailwind CSS v4**, **design tokens**, and **Framer Motion**.

All UI generated by the agent must follow these rules.

---

# Core UI Stack

Frontend UI is built using:

- React 19
- Next.js App Router
- Tailwind CSS v4
- Framer Motion
- shadcn/ui patterns

Reusable components are located in:

components/ui

Layout primitives are located in:

components/layout

Animation helpers are located in:

components/animations

Design tokens are defined inside:

design/tokens

---

# RTL Layout

This project is **Persian-first** and uses **RTL layout**.

All UI must support RTL properly.

Rules:

- Avoid assumptions about left-to-right layouts
- Use logical spacing utilities when possible
- Layout must remain correct in RTL environments
- Text alignment should default to `text-right`

---

# Tailwind Usage Rules

Styling must use **Tailwind CSS utilities**.

Guidelines:

- Prefer utility classes
- Avoid inline styles
- Avoid custom CSS unless absolutely necessary
- Prefer composition of utilities

Bad example:

style={{ padding: "16px" }}

Good example:

p-4

---

# Design Tokens

Design tokens are defined in:

design/tokens

Available token categories:

- colors
- typography
- shadows
- radius
- blur
- motion

Agent should always prefer tokens instead of arbitrary values.

Example:

Prefer:

rounded-lg

Instead of:

rounded-[13px]

---

# Typography

Fonts are located in:

public/fonts

Primary Persian font:

Kalameh

Typography tokens are defined in:

design/tokens/typography.ts

Guidelines:

- Use consistent text scale
- Avoid arbitrary font sizes
- Prefer predefined typography tokens

---

# Layout Components

Layout primitives are located in:

components/layout

Examples:

LayoutShell  
Sidebar  
SidebarShell  
SidebarContent  
Footer  

These should be used for structural layout.

Avoid rebuilding layout primitives.

---

# Animation System

Animations use **Framer Motion**.

Animation utilities exist in:

components/animations

Examples:

FadeIn  
SlideIn  
MotionSection  

Rules:

- Prefer existing animation helpers
- Avoid writing complex motion logic in pages
- Keep animations subtle and performant
- Avoid heavy animations that block rendering

---

# UI Composition Pattern

Preferred composition pattern:

Page  
→ Feature Component  
→ UI Components

Example:

app/blog/page.tsx  
→ features/blog/components/BlogGrid  
→ components/ui/Card

---

# Responsiveness

UI must support responsive layouts.

Use Tailwind responsive utilities:

sm:
md:
lg:
xl:

Example:

grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3

Avoid fixed widths unless necessary.

---

# Accessibility

All UI should follow basic accessibility rules.

Examples:

- buttons must use `<button>`
- forms must have labels
- interactive elements must be keyboard accessible
- aria attributes when necessary

---

# Performance

Prefer:

- lightweight components
- server components when possible
- minimal client-side JS

Avoid:

- unnecessary client components
- large animation libraries
- deeply nested DOM structures

---

# Agent UI Generation Rules

When generating UI:

1. Check existing UI components first
2. Reuse components from `components/ui`
3. Follow Tailwind utility patterns
4. Respect RTL layout
5. Use design tokens
6. Use Framer Motion helpers when animation is needed

Never introduce a new UI pattern if an existing one already exists.
